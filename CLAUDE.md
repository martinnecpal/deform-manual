# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A bilingual documentation site for **DEFORM V14.0.2** (a metal-forming FEM simulation package). The site is undergoing a migration from a static RoboHelp-generated HTML site to a **Jekyll 4.3** site for GitHub Pages.

- **Branch `jekyll-migration`** — active Jekyll rebuild (the current working state)
- **Branch `main`** — original RoboHelp static HTML; GitHub Pages serves from here

## Running Jekyll Locally

```bash
# Standard serve (baseurl = /HTML-DEFORM-manual, mirrors GitHub Pages)
bundle exec jekyll serve

# Dev mode (baseurl = "", so links work at localhost:4000/)
bundle exec jekyll serve --config _config.yml,_config_dev.yml

# Build only
bundle exec jekyll build
```

Output goes to `_site/`. The `_config_dev.yml` overrides `baseurl: ""` so asset and page links resolve correctly without the repo-name prefix.

## Jekyll Architecture

### Collections & Layouts
- `_docs/en/` and `_docs/sk/` — source content as Markdown files (migrated from `.htm`)
- `_config.yml` declares a `docs` collection; pages under `_docs` auto-get the `topic` layout
- `_layouts/default.html` — site shell (sidebar, topbar, footer, JS)
- `_layouts/topic.html` — wraps `default`, adds `<article>` wrapper for content pages
- `_includes/sidebar.html` — recursive Liquid partial; renders nested nav from a YAML data file
- `_includes/lang-switcher.html` — language toggle injected into `default` layout

### Navigation Data
- `_data/nav_en.yml` and `_data/nav_sk.yml` — hand-authored YAML trees that define the sidebar TOC
- Each entry has `name`, `url`, and optional `children` list
- When adding or renaming a page, update the matching nav YAML manually

### Search
- `search-index.json` — generated by Jekyll at build time; lists all doc pages with title and content
- `assets/js/search.js` — client-side search; lazy-loads the index on first focus, filters by substring match
- `assets/js/sidebar.js` — handles sidebar toggle and active-item expand/collapse

### Styling
- `assets/css/main.css` — primary stylesheet for the Jekyll layout
- `assets/css/lang-switcher.css` — language switcher styles (shared between old and new site)
- `assets/css/default.css` — base content typography
- Per-topic CSS files under `assets/css/<section>/` were generated by RoboHelp; still referenced by old `.htm` files but not by the Jekyll-migrated Markdown files

## Bilingual Structure

| Path | Purpose |
|------|---------|
| `index.md` | English home (Jekyll entry point) |
| `sk/index.md` | Slovak home (Jekyll entry point) |
| `_docs/en/` | English Markdown content |
| `_docs/sk/` | Slovak Markdown content |
| `_data/nav_en.yml` | English sidebar TOC |
| `_data/nav_sk.yml` | Slovak sidebar TOC |

The `default` layout detects language via `page.lang` front matter (`en` or `sk`) and renders the matching nav data file.

## Legacy Static Files (still present)

- `documentation_content/` — original RoboHelp `.htm` files; excluded from Jekyll build via `_config.yml`
- `template/` — RoboHelp theme (minified). **Do not edit minified files.** Excluded from Jekyll build.
- `whxdata/` — RoboHelp TOC/search data. Excluded from Jekyll build.
- `assets/equations/`, `assets/icons/`, `assets/images/` — shared image assets used by both legacy `.htm` files and migrated Markdown

## Deployment

GitHub Pages deploys from the `main` branch. The `baseurl` in `_config.yml` must match the GitHub Pages repo path (`/HTML-DEFORM-manual`). No CI/CD pipeline — pushing to `main` triggers a rebuild.

## Content Conventions

- Doc files use Markdown (`.md`) with front matter: `title`, `lang`, and optionally `layout`
- Numbering follows DEFORM manual chapters: `_docs/en/pre_processor/9_simulation_controls/9_1_simulation_type_settings.md`
- Images are still in `assets/images/` and `assets/equations/`; reference them with `{{ '/assets/images/foo.png' | relative_url }}` in Markdown or layouts
