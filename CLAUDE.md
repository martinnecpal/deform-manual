# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A static HTML documentation site for **DEFORM V14.0.2** (a metal-forming FEM simulation package), generated by **Adobe RoboHelp 2019** and hosted on GitHub Pages. There is no build step — the repository *is* the deployable output.

## Serving Locally

Open `index.html` directly in a browser, or use any static file server:

```bash
# Python
python -m http.server 8080

# Node (if http-server is installed)
npx http-server . -p 8080
```

The Slovak version is served from `sk/` (e.g. `http://localhost:8080/sk/`).

## Bilingual Structure

The site supports English (root) and Slovak (`sk/` subdirectory):

| Path | Purpose |
|------|---------|
| `index.html` | English entry point |
| `sk/index.html` | Slovak entry point |
| `documentation_content/` | English `.htm` content pages |
| `sk/documentation_content/` | Slovak `.htm` content pages |

The `sk/` version shares the root's `template/` and `whxdata/` directories (referenced via relative paths `../template/` etc.).

The language switcher is a custom addition injected into both `index.html` files. Its styles live in `assets/css/lang-switcher.css` and the HTML block looks like:

```html
<div class="lang-switcher">
  <a href="./" class="active"><span class="flag">🇬🇧</span> EN</a>
  <a href="./sk/"><span class="flag">🇸🇰</span> SK</a>
</div>
```

In `sk/index.html` the `href` values flip: `../` for EN and `./` for SK.

## Key Directories

- `template/` — RoboHelp theme (minified JS/CSS, images). **Do not edit minified files.**
- `template/sftc_suggest_deform/` — The active custom theme (layout CSS, logo, icons).
- `assets/css/` — Per-section CSS files, one per `.htm` topic. Also `lang-switcher.css`.
- `assets/images/` — Screenshots referenced by content pages.
- `assets/equations/` — Equation images (`.jpg`) referenced inline in content.
- `assets/icons/` — UI icon images used in content.
- `whxdata/` — RoboHelp-generated TOC (`toc*.new.js`), search index (`text/*.js`), glossary (`glo.new.js`). These are auto-generated — regenerate via RoboHelp if TOC changes.
- `documentation_content/` — All English topic pages as `.htm` files, organized by numbered sections matching the DEFORM manual chapters.

## Content Page Conventions

- All topic files use the `.htm` extension.
- Numbering follows DEFORM manual chapters: `9_simulation_controls/9_1_simulation_type_settings.htm`.
- Each topic typically has a matching CSS file under `assets/css/` with the same path structure.
- Images are referenced relative to the topic file, usually pointing to `assets/images/...` from the repo root.
- When adding a new page, you also need to update `whxdata/toc*.new.js` and the sitemap (`sitemap.xml`) — or regenerate via RoboHelp.

## Deployment

The site deploys to GitHub Pages directly from the `main` branch. The `sk/` subfolder is served as a sub-path automatically. No CI/CD pipeline exists — pushing to `main` triggers GitHub Pages to serve the updated content.
