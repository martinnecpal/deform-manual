Abstract

Enhanced recrystallization and grain growth kinetics at mesoscale levels [1-6] are now available in DEFORM 12.1 cellular automata (CA) models. CA model can be run in post processing mode in either Material Suite or Next Generation (NG) postprocessor at user selected locations and a completed nominal 2D or 3D database is a prerequisite to run DEFORM CA model. A variety of mechanisms can be considered, including dynamic recrystallization (DRX), metadynamic recrystallization (MRX), static recrystallization (SRX) and static grain growth (SGG). Three typical crystal structures for metallic materials (FCC, BCC, and HCP) can be considered, and the misorientation angle between neighboring grains is calculated in terms of crystal symmetry. User routines and user CA models are supported. The material can be single-phase, dual-phase, or multiple-phase. The initial microstructure for CA model can be generated by DEFORM based on the average grain size, the deviation of grain size, aspect ratio, phase volume fraction, and the distribution of secondary phase. Importing EBSD data for the initial microstructure of CA model is supported. For the dual-phase or multiple-phase material with the secondary phase recrystallization, currently the user should implement their own CA model via user routines. 

The basic input data for the CA model include the Thermo-mechanical Process model (TMP) point-tracking data (time, step, strain, strain rate, temperature, deformation gradient, and velocity gradient), material parameters and the initial microstructure. For each saved step in the nominal database, the output from CA model include the detailed microstructure such as average dislocation density, flow stress, the number of grains, the average grain size, and the recrystallization volume fraction. The variation of flow stress, average grain size, and recrystallization volume fraction with strain (or time) can be plotted. 

In this lab, the dynamic recrystallization during forging and the static grain growth during annealing of nickel-based superalloy IN718 are simulated by CA model in DEFORM Material Suite.

## Launch CA model in material suite

On a Windows machine, go to the ![]({{ '/assets/icons/pre_icons/windows_start.jpg' | relative_url }}) button, select DEFORM-v12.xxx (.xxx indicates version number E.g. v14.0.2 and select DEFORM GUI Main v1x.xx from the menu. The DEFORM GUI Main window will appear.  
  
We require a nominal DB to perform CA simulation in material suite, for this lab we will use **Forging_IN718_Nominal.key** from **\MANUALS\PDF \DEFORM_CA_Lab** folder to generate nominal DB. The operation is a uniform compression of an IN718 billet at 1050ºC at a strain rate of 1/s. Open NG Pre using the link DEFORM GUI Main and define Problem ID as "**Forging_IN718** ", select "**2D** " Dimension radio button and "**System International (SI)** " unit System radio button in New Problem window, then import **Forging_IN718_Nominal.key** from **\MANUALS\PDF \DEFORM_CA_Lab folder**. Go to Generate DB page and click on " Generate Database " to generate database. Switch to “Simulation” Tab and run the simulation, make sure simulation stops normally without any issue. Now from GUI Main click on **Material Suite** under “Post Processor” to open Material Suite module. User can also double click "DEF_GUI_MAT.exe" in the installation directory of DEFORM software to launch material suite.  
  
After the main window for material suite pops up, click "Open..." icon ( ![]({{ '/assets/icons/pre_icons/mo_open_icon.jpg' | relative_url }}) ) in the top tool bar to import the example database (**Forging_IN718.DB**). The default task item "DEFORM" is added to the task tree. Expand "Microstructure" item and click "Grain Model CA" icon, the new task item "CA model" is added to the task tree, as shown in Fig. L1.1.

For “import from experimental EBSD data“ method, click “EBSD Data Setup” button, the window for EBSD data setup pops up, as shown in Fig. L1.10. (only for demonstration purpose, not required for this Lab). The EBSD data should follow the specified format. Each line in header stars with “#”. Each data line can have 9~14 numerical fields. The first 9 fields must be arranged as:

Phi1, PHI, Phi2, X, Y, IQ, CI, Phase ID, grain ID, …

Euler angles are in Bunge’s notation in radians. Each data line is assumed to be a valid scanning point.

Besides data format requirement, we need to specify grid type (square or hexagon), the number of scanning rows, the number of columns (or the number of columns for odd and even rows for hexagon grid), scanning step size, the grain ID status, the number of phases, the crystal type of each phase, and the phase mapping relationship between EBSD data and CA model. The maximum phase number is 5. If grain ID is not determined in EBSD data, the sorting code will be called to assign grain ID. After finishing EBSD data setup, click the import button (![]({{ '/assets/icons/pre_icons/mo_open_icon.jpg' | relative_url }})) to load EBSD data and display grain microstructure image, as shown in Fig. L1.11. (only for demonstration purpose, not required for this Lab). When the EBSD scanning area is larger than the CA domain size, we can select a desired region in image by moving the cropping window, then click “crop” button.

For flow stress and grain boundary energy, the user routines are supported (SUBROUTINE USR_FLOWSTRESS and SUBROUTINE USR_UGBE). For user CA model (e.g. SUBROUTINE USR_CA_DRX), user-defined internal state variables for each cell are supported. The user-defined internal state variables should be initialized at the beginning of first step and update each step via user routine (SUBROUTINE USR_UISV). The usage of user routines will be introduced at the end of section. 

The next tree item is “**Dislocation****density during deformation** ” and the corresponding data window is shown in Fig. L1.14. The dislocation density is an important internal state variable for each cell in CA-DRX model, because it determines the nucleation and grain growth, as well as flow stress. We provide three dislocation density models, i.e., modified Yoshie–Lassraoui–Jonas (YLJ-GM) model, Goetz-Seetharaman (GS) model, and Kocks-Mecking (KM) model. User routine for dislocation density evolution is also supported (SUBROUTINE USR_DRX_DISDEN). In this Lab, we select YLJ-GM model, and define the following material parameters: reference strain rate = 1/s, h0 = 32.1917 um-2, mh = 0.0066, Qh = 75212.8 J/mol, r0 = 1713.79, mr = 0.221256, Qr = 31721.4 J/mol.

In this Lab, select the radio button “By critical strain” and set the following material parameters: ![]({{ '/assets/images/labs/material_suite_labs/material_suite_labs_for_ca_model/epsolon_c_0.jpg' | relative_url }}) = 0.000349, mc = 0.157, Qc = 63433 J/mol (see Fig. L1.15.).

The next tree item to define is the “**Nucleation****rate** ” and the corresponding data definition window is shown in the Fig. L1.16. There are two options for the calculation of nucleation rate: Generalized model or User routine (SUBROUTINE USR_DRX_NUCRAT ). In this Lab, we select the “Generalized model” radio button, and input the material parameters as follows: cn = 13.18, mn = 1, Qn= 52793.1 J/mol. 

![]({{ '/assets/images/labs/material_suite_labs/material_suite_labs_for_ca_model/cal1_image0017.jpg' | relative_url }})

Nucleation rate

The last tree item to define is the “**Grain****growth** ”, as shown in Fig. L1.17. A generalized model applicable for both recrystallization and static grain growth is implemented to calculate the grain boundary migration rate. The driving force for grain growth (![]({{ '/assets/images/labs/material_suite_labs/material_suite_labs_for_ca_model/delta_p.jpg' | relative_url }})) consists of four terms (PD, PC , PZ, and PS). P D is the driving force contributed by the dislocation density difference between two neighboring grains, which plays a dominant role during recrystallization (DRX, MRX, and SRX). PC is the driving force to reduce the grain boundary energy, which plays a dominant role during static grain growth (PD = 0). ![]({{ '/assets/images/labs/material_suite_labs/material_suite_labs_for_ca_model/aplha_k.jpg' | relative_url }}) is a correction factor to grain boundary curvature, which is calibrated using static grain growth data. PZ considers the Zener pinning effect and PS the solute drag effect. User routine for grain growth is also supported (SUBROUTINE USR_LOC_VEL). In this Lab, we select the “Generalized model” radio button, and set ![]({{ '/assets/images/labs/material_suite_labs/material_suite_labs_for_ca_model/delta_d_0.jpg' | relative_url }}) = 1.15954e-6 m3/s, Qm = 330008 J/mol, ![]({{ '/assets/images/labs/material_suite_labs/material_suite_labs_for_ca_model/aplha_k.jpg' | relative_url }}) = 0.0048, A = 0, z1 = 0.0. The other material parameters use the default values.

![]({{ '/assets/images/labs/material_suite_labs/material_suite_labs_for_ca_model/cal1_image0018.jpg' | relative_url }})

Grain growth rate

## Output

After simulation is finished, all the selected points are listed under the "**Output** " item in the task tree. Specify a step number in the step browser and select a point, the simulation results can be reviewed, as shown in Fig. L1.20. Various aspects of the microstructure can be displayed by clicking the different display options, including the grain morphology (represented by grain id), grain boundaries, as well as the dislocation density. In addition, statistical information from the simulated microstructure are also displayed, including the histograms of grain size, misorientation angle and grain aspect ratios. The min, max, average, and standard deviation values are shown as well. The curves, such as the variation of flow stress, average dislocation density, average grain size, recrystallization volume fraction with strain or time, can also be displayed.
