DEFINITION
BCCTMP specifies the heat transfer boundary constraint code for individual nodes.

 

REMARKS

The boundary constraint code (bcc) options for DefBCHeat and BCHeat(i) are:

0 nodal heat

1 nodal temperature

2 node is exchanging heat with the environment via convection and radiation 

3 nodal heat flux 

-n node is in contact with object n

 

For bcc = 0, the value of the nodal heat should be specified with NDHEAT or BCCFNC.

For bcc = 1, the value of the nodal temperature should be specified with NDTMP or BCCFNC.

For bcc = 2, convection and radiation heat transfer calculations will be performed at the specified node.

For bcc = 3, the value of the nodal heat flux should be specified with NDFLUX or BCCFNC.

For bcc = -n, conduction heat transfer calculations between object Object and object n will be performed at the specified node. For non-isothermal simulations, contact boundary constraint are updated by DEFORM during the simulation to account for nodal contact and separation associated with the object's deformation.

 

Contact constraints are only applied to slave objects.

 

Each node may only be assigned one type of boundary constraint code.

 

Applicable object types:  Rigid, Elastic, Plastic, Elastoplastic, and Porous.
